# v0.2.0 Design: Category Hierarchy, Bridging, EmbeddingProvider, MCP Extensions

**Date:** 2026-03-07
**Status:** Approved

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Hierarchy depth | Single `parent_id`, no depth constraint | Algorithm self-limits; suppressing valid 3rd-level signals wastes information |
| Split trigger | Member count >= 8 AND coherence < 0.6 | Prevents premature splitting of small/coherent clusters |
| Bridging mechanism | Explicit `MemberOf` links with `bridging_weight = 0.3` | Consistent graph model; spread_activation needs zero changes |
| EmbeddingProvider | Sync trait, `embed()` required + `embed_batch()` default | Standard pattern; batch override for performance-sensitive providers |
| MCP tools | All 5 (3 new + 2 extended) | Low cost, complete API surface |

## 1. Schema Changes

Schema version: 3 → 4.

**Categories table** — add column:
```sql
ALTER TABLE categories ADD COLUMN parent_id INTEGER REFERENCES categories(id);
```

**Links table** — new `LinkType::MemberOf` for category bridging. Created when
nodes are assigned to categories, deleted on reassignment or category deletion.

No new tables. EmbeddingProvider is a trait, not storage.

## 2. Category Hierarchy + Evolution

### Hierarchy

- `store_category()` gains optional `parent_id` parameter
- `list_categories()` returns all; agent builds tree via `parent_id`
- New `get_subcategories(conn, parent_id)` returns direct children
- Label inherits from prototype node's content (unchanged)

### Splitting

Runs during `maintain_categories()` in `transform()`.

**Trigger:** `member_count >= SPLIT_MIN_MEMBERS (8)` AND average cosine
similarity of members to centroid < `SPLIT_COHERENCE_THRESHOLD (0.6)`.

**Procedure:**
1. Re-run `discover_categories` on the category's members only
2. Sub-clusters get `parent_id` pointing to the original category
3. Original becomes a container; direct members stay, sub-category members
   transfer to children
4. If re-clustering produces no sub-clusters (still coherent), skip

### Merging (updated)

Existing merge (centroid cosine > 0.85) stays. When merging categories with
different parents, the merged category takes the parent of the larger one.

### Constants

```rust
const SPLIT_MIN_MEMBERS: usize = 8;
const SPLIT_COHERENCE_THRESHOLD: f32 = 0.6;
```

## 3. Cross-Domain Bridging

### MemberOf Links

- `assign_node_to_category()` creates bidirectional `MemberOf` links:
  - `Semantic(node_id) → Category(category_id)` weight = 0.3
  - `Category(category_id) → Semantic(node_id)` weight = 0.3
- On reassignment: delete old links, create new ones
- On category deletion: cascade delete all MemberOf links

### Spreading Activation

No code changes. Activation flows naturally:

```
Episode → (co-retrieval) → Semantic → (MemberOf 0.3) → Category → (MemberOf 0.3) → other Semantic
```

Signal strength at cross-category destination: ~0.18 (0.3 × 0.6 decay).
Weak enough to not dominate, strong enough to surface related concepts.

Category-to-category links deferred to v0.3.

## 4. EmbeddingProvider Trait

```rust
pub trait EmbeddingProvider {
    fn embed(&self, text: &str) -> Result<Vec<f32>>;

    fn embed_batch(&self, texts: &[&str]) -> Result<Vec<Vec<f32>>> {
        texts.iter().map(|t| self.embed(t)).collect()
    }
}
```

### Integration

- `AlayaStore::set_embedding_provider(&mut self, provider: Box<dyn EmbeddingProvider>)`
- `store_episode()`: auto-embed if `episode.embedding` is `None` and provider set
- `consolidate()`: auto-embed semantic nodes without embeddings
- `query()`: auto-embed `query.text` if `query.embedding` is `None` and provider set
- When no provider: behavior identical to v0.1

### Why set_embedding_provider() not constructor

- Keeps `open()` / `open_in_memory()` simple
- Provider can be set after construction
- Matches explicit-lifecycle philosophy

## 5. MCP Tool Extensions

| Tool | Change | Parameters |
|------|--------|------------|
| `categories` | NEW | `min_stability: Option<f32>` |
| `knowledge` | EXTEND | Add `type`, `min_confidence`, `category_id`, `limit` |
| `recall` | EXTEND | Add `boost_category: Option<i64>` |
| `neighbors` | NEW | `node_type`, `node_id`, `depth` (default 1) |
| `node_category` | NEW | `node_id: i64` |

All follow existing `#[tool]` pattern. Return human-readable strings.

## 6. Testing Strategy

TDD: every feature starts with a failing test.

| Feature | Tests |
|---------|-------|
| Schema v3→v4 migration | `parent_id` exists, idempotent |
| Category hierarchy | store with parent, get_subcategories, tree |
| Category splitting | Trigger conditions, sub-categories get parent_id |
| MemberOf links | Create/delete on assign/reassign/category delete |
| Cross-domain bridging | Activation reaches nodes through category |
| EmbeddingProvider | Auto-embed episode/query/consolidation, no-op unset |
| MCP categories | Formatted list, min_stability filter |
| MCP knowledge filter | Type, confidence, category, limit |
| MCP recall boost | Category bias affects ranking |
| MCP neighbors | Graph neighbors with weights |
| MCP node_category | Returns category or uncategorized |
| Proptest | Bridging weight [0,1], coherence [0,1] |

Estimated: ~25-30 new tests (201 → ~230).
