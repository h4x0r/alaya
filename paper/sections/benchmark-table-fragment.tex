\smallskip
\noindent\textbf{Methodological caveats.}\enspace
Comparing scores across rows requires care for several reasons:

\begin{enumerate}[leftmargin=*,nosep]

\item \textbf{Judge LLM variation.}
The LoCoMo LLM-as-Judge protocol requires a ``judge'' model to score each answer against a gold reference.
The original benchmark code defaults to \texttt{gpt-4o-mini}, but some evaluators use \texttt{gpt-4o}, \texttt{gpt-4.1}, or \texttt{gpt-4-turbo} as the judge~\cite{maharana2024locomo}.
Since different judges exhibit different false-positive and false-negative rates, a score of 90\% with one judge is not directly comparable to 90\% with another.
No study has published a systematic judge-calibration analysis, though the Hindsight paper~\cite{latimer2025hindsight} uses a fixed judge across all baselines.

\item \textbf{Self-reported vs.\ independently verified scores.}
The most dramatic illustration is Zep on LoCoMo:
Zep originally self-reported 84.61\%~\cite{zep2025},
later corrected to 75.14\%~\cite{zep2025};
Mem0 evaluated Zep at 65.99\%~\cite{choudhary2025mem0};
and Mem0's subsequent standardized replication yielded 58.44\%~\cite{mem0replication2025}.
This 26-percentage-point spread for the \emph{same system} underscores the fragility of reported numbers.
Similar concerns apply to Backboard's self-reported 90.1\% on LoCoMo, which the Hindsight authors note ``could not be independently reproduced''~\cite{latimer2025hindsight}.

\item \textbf{LoCoMo's limited dialogue length.}
LoCoMo conversations average 16--26K tokens~\cite{maharana2024locomo}, well within the 128K--1M context windows of modern frontier models.
A simple full-context baseline (feeding the entire conversation to GPT-4o) achieves ${\sim}$73\%, outperforming several dedicated memory systems.
This ceiling compression limits LoCoMo's discriminative power for systems designed to handle much longer histories.
LongMemEval\_S (${\sim}$115K tokens per question) and LongMemEval\_M (${\sim}$1.5M tokens) provide a harder test~\cite{wu2025longmemeval}.

\item \textbf{DMR's near-saturation.}
The DMR benchmark's 60-message conversations fit within a single context window, and the full-context baseline (94.4\%) already rivals or exceeds most memory systems.
With Zep reaching 98.2\%, DMR is effectively saturated and no longer discriminates among modern approaches~\cite{dempsey2025graphiti}.

\end{enumerate}

\smallskip
\noindent\textbf{The benchmark-wars problem.}\enspace
The agent memory field is experiencing an acute form of what might be called \emph{benchmark wars}:
vendors routinely dispute each other's evaluation results, and scores for the same system
can vary by 20+ percentage points depending on who runs the evaluation.
Mem0 and Zep have publicly traded accusations of implementation errors, prompt tampering,
and category-inclusion mistakes in each other's LoCoMo evaluations~\cite{zep2025,choudhary2025mem0,mem0replication2025}.
Backboard, EverMemOS, and MemMachine each claim ``state-of-the-art'' on LoCoMo using
subtly different judge models, prompt templates, and answer-generation backends,
making direct comparison unreliable.
On LongMemEval, score inflation is driven partly by model scaling---Mastra's Observational
Memory jumps from 84.23\% (GPT-4o) to 94.87\% (GPT-5-mini)---blurring the line between
\emph{memory architecture} improvements and \emph{backbone LLM} improvements.
The MEMTRACK benchmark (NeurIPS 2025 Workshop) found that bolting Mem0 or Zep onto
GPT-5 or Gemini-2.5-Pro yielded no statistically significant improvement over the bare
model~\cite{memtrack2025}, suggesting that for frontier-scale LLMs, the value of external
memory layers may depend heavily on the deployment context (history length, latency budget,
cost constraints) rather than on benchmark accuracy alone.
Until the community adopts standardized evaluation protocols---fixed judge models,
frozen prompt templates, mandatory reproduction packages, and blind third-party
evaluation---published benchmark tables should be interpreted with substantial caution.
